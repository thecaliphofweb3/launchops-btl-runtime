# Submission Checklist

Submit by **Jul 5, 2026 at 15:00 UTC**.

## Include

- Public or judge-accessible repository.
- Setup instructions.
- Two-minute demo video.
- Short explanation of the user problem.
- Short explanation of how the project uses the BTL runtime.
- Team member names and contact emails.

## Demo structure

1. Show the problem in 15 seconds.
2. Show the working product in 75 seconds.
3. Show the BTL runtime call in 20 seconds.
4. End with what you would build next in 10 seconds.

## Judging reminders

- Runtime use matters.
- A working narrow product beats a huge unfinished one.
- Judges should understand the value without reading your whole codebase.
