# Organizer Runbook

## Before kickoff

- Confirm registration export and email list.
- Prepare scoped BTL runtime keys and starter credits.
- Create Discord channels for announcements, team formation, runtime help, demos, and off-topic chat.
- Pin the runtime quickstart, submission checklist, and judging rubric.
- Confirm mentor coverage across major time zones.

## Kickoff

- Welcome everyone.
- Restate the single rule: every project must call the BTL runtime.
- Walk through the starter call.
- Explain prizes, submission format, demo limits, and the hard deadline.
- Send API keys and credit details.

## During hacking

- Keep runtime support visible.
- Collect common issues and post fixes in announcements.
- Encourage teams to narrow scope by midpoint.
- Ask teams to post a one-line project update before demo day.

## Submission review

- Check repo access.
- Check demo video length.
- Check runtime usage proof.
- Queue finalist demos before the live call.

## After winners

- Confirm payout method with each winner.
- Send certificates.
- Invite strong teams into BTL beta follow-up sessions.
- Publish a recap with project links.
